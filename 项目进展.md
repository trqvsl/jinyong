# 金庸群侠传 · 项目进展

> 最后更新：2026-06-30
> 这是一个致敬"半瓶神仙醋"版金庸群侠传2/3 的网页小游戏。

---

## 一、项目目标

做一个**数值养成 + 事件驱动 + 轻量回合制战斗**的武侠网页游戏，参考半瓶神仙醋版金庸群侠传的核心乐趣：创建角色 → 修炼/拜师/闯荡 → 武功搭配 → 变强通关。

### 设计原则
- **数据驱动**：武功、门派、敌人都拆成独立数据文件，加内容只动数据不碰核心逻辑。
- **长期可迭代**：用 TypeScript 类型系统防 bug，存档做向后兼容迁移。
- **随时随地玩**：纯前端单机游戏，部署到 Cloudflare Pages 给链接就能玩，存档在浏览器本地。

---

## 二、技术栈

| 技术 | 用途 |
|------|------|
| Vite | 构建工具 / 开发服务器 |
| React | UI 框架（组件化） |
| TypeScript | 类型安全（数据密集型游戏的必需品） |
| 纯 CSS | 样式（含战斗动画） |

**部署方案**：开发完构建成静态文件，扔到 Cloudflare Pages / Vercel / GitHub Pages 即可，零服务器成本。
**存档方案**：localStorage（本地存档）+ 存档码导入导出（跨设备）。

## 六、已知的技术要点（供下次接手）

1. **写文件必须用 UTF-8 无 BOM**：PowerShell 的 Set-Content 默认带 BOM 会导致 JSON 解析失败。项目里用 `[System.IO.File]::WriteAllText($path, $content, (New-Object System.Text.UTF8Encoding $false))` 写文件。
2. **旧存档迁移**：player.ts 的 migratePlayer() 会给旧版武功补 category 字段。以后改数据结构都要在这里加兼容。
3. **React 闭包陷阱**：BattleScreen 用 useRef 持有最新状态，避免 setTimeout 读到旧值。改战斗逻辑时注意这个。
4. **启动开发服务器**：`npm run dev`，访问 http://localhost:5173/
5. **重新安装依赖**：换了电脑后先 `npm install`，再 `npm run dev`。

## 七、Git 版本管理（首次配置）

### 安装 Git
- **Windows**：去 https://git-scm.com 下载安装，或 `winget install Git.Git`
- **Mac**：终端运行 `xcode-select --install` 或 `brew install git`

### 首次初始化（在项目目录执行）
```bash
git init
git add .
git commit -m "金庸群侠传：MVP 完成（四类武功+状态系统+门派+战斗动画）"
```

### 推送到 GitHub（用于跨设备同步）
1. 在 GitHub 新建一个空仓库（比如叫 jinyong）
2. 按 GitHub 提示执行：
```bash
git remote add origin https://github.com/你的用户名/jinyong.git
git branch -M main
git push -u origin main
```

### 换电脑继续开发（如 Mac）
```bash
git clone https://github.com/你的用户名/jinyong.git
cd jinyong
npm install        # 安装依赖
npm run dev        # 启动开发服务器
```
> 注意：Mac 上用 `npm install` 而不是 Windows 的命令，完全通用。

### 日常提交习惯
每次开发完一个功能：
```bash
git add .
git commit -m "描述这次改了什么"
git push
```
